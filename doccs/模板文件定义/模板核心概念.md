# 模板核心概念

## 模板数据结构

模板文件采用 JSON 格式存储，主要包含以下核心字段：

- **TemplateId**: 模板唯一标识符
- **Name**: 模板名称
- **Description**: 模板描述
- **CreatedDate**: 创建时间
- **ModifiedDate**: 修改时间
- **Layout**: 布局信息，包含纸张类型、尺寸、边距以及元素集合
- **Config**: 配置信息，包含版本、作者等
- **ParentId**: 父模板ID（树结构）
- **ChildTemplateIds**: 子模板ID列表（树结构）

## 布局信息 (Layout)

布局信息包含以下字段：

- **PaperType**: 纸张类型（0=A4, 1=A5等）
- **IsLandscape**: 是否横向
- **PaperWidth**: 纸张宽度
- **PaperHeight**: 纸张高度
- **MarginLeft**: 左边距
- **MarginRight**: 右边距
- **MarginTop**: 上边距
- **MarginBottom**: 下边距
- **FixedElements**: 固定元素集合
- **EditableElements**: 可编辑元素集合

## 元素类型

模板中的元素分为固定元素和可编辑元素，支持以下类型：

- **TextBox** (1): 文本框
- **FixedText** (0): 静态文本
- **ComboBox** (5): 下拉框
- **DatePicker** (6): 日期选择器
- **CheckBox** (7): 复选框
- **Line** (2): 直线
- **Rectangle** (8): 矩形
- **Circle** (9): 圆形/椭圆
- **Image** (10): 图片
- **Table** (4): 表格
- **Chart** (11): 图表

## 模板服务

模板服务 (`ITemplateService`) 提供以下核心功能：

- 模板的增删改查
- 模板的导入导出
- 模板的版本管理
- 报告实例的管理
- 模板树结构的获取

## 模板文件存储

模板文件存储在以下目录：

- **baseTemplate**: 基础模板目录
- **Templates**: 实际模板目录
- **Versions**: 版本目录
- **Reports**: 报告实例目录

## 模板加载流程

1. 应用启动时初始化模板服务
2. 扫描基础模板目录和实际模板目录
3. 加载默认模板（优先加载 A4 竖向生化模板）
4. 提供模板树结构给 UI 展示
5. 用户选择模板后加载到编辑器

## 模板保存流程

1. 编辑器中修改模板内容
2. 调用保存方法，更新修改时间
3. 将模板数据序列化为 JSON
4. 写入对应目录的 JSON 文件
5. 更新模板缓存
